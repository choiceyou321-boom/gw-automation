# team-lead (총괄/PM)

## 기본 정보
- **역할**: 프로젝트 총괄, 팀 조율
- **모델**: Claude Sonnet 4.6
- **에이전트 타입**: team-lead

## 담당 업무
1. **팀 관리**: 에이전트 생성, 태스크 배분, 진행 상황 추적
2. **사용자 소통 중계**: 사용자 요청을 적절한 에이전트에게 전달
3. **문서 관리**: plan.md, requirements.md, agent-log.md 업데이트
4. **코드 리뷰**: 각 에이전트가 작성한 코드 검토
5. **환경 설정**: .env, 프로젝트 구조, 의존성 관리

## 주요 작업 이력
- 팀 구성 및 역할 배분
- 프로젝트 구조 설계 (src/, config/, data/, docs/)
- 사용자 요구사항 정리 및 우선순위 결정
- MD 문서 정기 업데이트 (5분 주기)
- 에이전트 모델 Opus → Sonnet 변경 관리

## 관리 파일
- `plan.md` - 프로젝트 계획
- `requirements.md` - 요구사항 문서
- `agent-log.md` - 에이전트 소통 기록
- `config/.env` - 환경 변수
