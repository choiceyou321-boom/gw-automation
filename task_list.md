## 🚀 1. 가장 시급한 핵심 과제 (우선순위 1순위)

### [x] Task 1. 임시보관문서 열기 및 결재상신 E2E 테스트 완성 ✅ (세션 VI 완료)
- 상세 내용:
  현재 사용자 그룹웨어의 임시보관문서함에 6건의 '[프로젝트]지출결의서' 데이터가 저장된 상태입니다.
  (API 로 확인 완료)
  실제 브라우저 자동화 봇(Playwright)을 조작해 이 중 1건의 양식을 팝업 화면으로 띄웁니다.
  가장 중요한 결재상신 처리 기능을 안정적으로 동작시키는지 테스트(end-to-end 테스트) 해야 합니다.
  (보관 버튼은 존재하지 않음을 확인)
- 💡 설명:
  - E2E(End-to-End) 테스트: 처음 로그인부터 최종 결재상신 버튼을 눌러보고 성공 메시지가 뜨는지에 이르는 전체 흐름을 사람의 개입 없이 실제 환경에서 완벽하게 진행해 보는 테스트입니다.
  - Playwright (플레이라이트): 브라우저 환경을 코드로 조작하여 마우스 클릭과 키보드 입력을 자동화 해주는 도구입니다.

### [x] Task 2. 지출내역 상세 항목(그리드, Grid) 및 첨부파일 자동 입력 고도화 ✅ (세션 VI 완료 — RealGrid API 방식)
- 상세 내용:
  현재 테스트 된 폼에서는 제목만 자동으로 입력되고 있습니다.
  지출결의서의 실제 내용인 용도명, 상세내용, 관련 부서 등, 공급가액 액수 등의
  추가 데이터 항목(Grid 행)을 완벽하게 자동으로 기입해야 합니다.
- 💡 설명:
  - 그리드(Grid): 엑셀의 표처럼 결재문서 중간에 여러 개의 항목을 넣어야 하는 다중입력 구조입니다. (예: 점심 식사 1건, 저녁 야근 1건 추가)
  - DOM 탐색 (Document Object Model): 문서를 구성하는 각 입력칸/버튼 등을 구조화한 지도에서 정확한 빈칸 위치(Selector 주소)를 찾아내 타이핑하게 하는 로직입니다.

### [x] Task 4. 결재 모듈 및 회의실 로그인 동작 보완 테스트 ✅ (세션 VI 완료)
- 상세 내용:
  기존 로그인 진행 시 비활성화(disabled) 된 #reqCompCd(회사코드 텍스트 박스) 필드에

  이후 회의실 예약 및 결재 메뉴 진입까지 이상 없이 이어지는지 재확인 테스트가 필요합니다.
- 💡 설명:
  - Fallback(폴백) 로직: 첫 번째 규칙으로 입력칸을 못 찾으면 대안의 다른 규칙으로 작동하게 만드는 일종의 안전망 코드입니다. 이번에 비활성화된 빈칸을 무시하도록 코드가 수정되었습니다.

### [x] Task 5. 필수 필드 누락 방지 및 에러 핸들링 추가 (결재 자동화) ✅ (세션 VI 완료)
- 상세 내용:

## 🔐 4. 인증 시스템/네트워크 마이그레이션 및 서비스 배포 준비

### [x] Task 9. 신규 wehago-sign (HMAC) 방식의 확장 적용 및 예외상황 처리 ✅ (세션 VI 완료 — 자동 재인증 구현)
- 상세 내용:
  그룹웨어 내부 회의실(자원)을 예약할 때 기존 Playwright 의 UI 조작 우회 방식을 거치지 않고,
  서명 암호학 코드인 HMAC 공식 해독을 통해 httpx API 호출 방식으로 전환을 성공했습니다.
  서명이 만료되거나 세션 토큰이 강제 끊겼을 시 쿠키를 재발급받는
  예외처리 시나리오를 고도화해야 합니다.
- 💡 설명:
  - HMAC(wehago-sign) 서명: 데이터가 변조되지 않게 암호화 키를 활용해 씌우는 도장입니다.
    더존 API는 이걸 요청마다 새롭게 찍기 때문에 이를 코드상에서 뚫는 게 가장 어려운 과정이었습니다.
  - httpx API 호출 (REST 방식): 브라우저를 실제 보이지 않게 띄워 마우스를 조작하는 대신,
    컴퓨터끼리 규칙된 서버 주소 및 코드로만 데이터를 직접 대화하듯 교환하는 방식으로 매우 빠릅니다.
