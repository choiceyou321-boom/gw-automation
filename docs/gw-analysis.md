## 7. 기술 구조

### 플랫폼
- **더존 Amaranth10** (WEHAGO 플랫폼)
- **React SPA** (Single Page Application)
- **Hash routing**: `/#/모듈코드/메뉴코드/페이지코드`
- 공개 API 없음 → 내부 REST API 사용 (POST 기반)
- schres 모듈: `/modules/schres/static/js/` (동적 로딩)

### URL 라우팅 규칙
```
/#/{모듈}/{메뉴}/{페이지}?specialLnb=Y&moduleCode={M}&menuCode={MC}&pageCode={PC}
```
- UB = 전자결재 모듈
- UK = 자원(회의실) 모듈
- UD = 메일 모듈

### 팝업 패턴
- 결재 문서 상세 → **새 창(팝업)으로 열림**
- URL: `/#/popup?MicroModuleCode=eap&docID={문서ID}&formId={양식ID}`
- Playwright에서 `context.on('page')` 이벤트로 캡처 필요

### 인증
- 2단계 로그인: ID 입력→엔터→PW 입력→엔터
- `#reqCompCd`: 회사코드 (disabled, "glowseoul")
- `#reqLoginId`: 아이디
- `#reqLoginPw`: 비밀번호
- 세션: cookie 기반 (storage_state로 저장/복원 가능)
- schres API: wehago-sign 동적 서명 (Playwright evaluate로 우회)
