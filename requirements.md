# 그룹웨어 자동화 요구사항 정리

> 최종 업데이트: 2026-03-01 21:30

## 그룹웨어 정보
- URL: https://gw.glowseoul.co.kr/#/
- 시스템: 더존 Amaranth10 (klago 플랫폼)
- 로그인 방식: ID/비밀번호 입력
- 자동화 방식: Python + Playwright (하이브리드)

---

## 요구사항 1: 결재 양식 분석 (내부 작업)
> ⚠️ 사용자용 기능 아님 - 개발팀이 자동화 설계를 위해 분석하는 작업

- 과거 상신한 전자결재를 분석하여 양식/패턴 파악
- 결재 양식 종류, 필드 구조, 결재선 패턴 수집
- Phase 5 (결재 자동 작성) 구현을 위한 사전 조사
- **사용자에게 엑셀 등으로 제공하는 기능 아님**

## 요구사항 2: 결재 문서 자동 작성
- 채팅으로 "경비 결재 올려줘" + PDF/이미지 첨부 → 자동 작성
- 지출/경비 청구 등 반복 양식 자동 작성
- 첨부 파일 자동 처리
- 빈도: 비정기적 (필요할 때)

## 요구사항 3: 안 읽은 메일 요약 → Notion 저장
- 그룹웨어의 안 읽은 메일을 자동으로 요약
- 요약 결과를 Notion 지정 위치에 정리
- **대기 중**: Notion API Key 재확인 + 저장 페이지 URL 미설정
- 빈도: 주기적

## 요구사항 4: 회의실 예약 자동화 ★ 1순위
- 채팅으로 "내일 오후 2시 회의실 예약해줘" → 자동 예약
- 빈 회의실 검색, 예약 등록
- 빈도: 필요할 때

## 요구사항 5: 챗봇 웹 인터페이스 ★ 핵심 요구사항
- **웹페이지 채팅창**에서 자연어로 자동화 작업 요청
- **파일 첨부**: 드래그앤드랍으로 이미지/PDF 첨부
- **AI 판단**: Claude Sonnet 4.6이 요청 분석 → 적절한 자동화 작업 실행
- **결과 알림**: 처리 결과를 채팅으로 응답
- **결과 보관**: 로컬 폴더에 파일 저장
- **접속**: 외부 어디서든 접속 가능 (클라우드 배포)

### 사용 시나리오 예시
1. "회의실 예약해줘, 내일 오후 3시" → 회의실 자동 예약
2. "이 영수증으로 경비 결재 올려줘" (PDF 드래그앤드랍) → 결재 양식 자동 작성
3. "안 읽은 메일 요약해줘" → 메일 수집/요약 결과 표시
4. "지난달 결재 내역 알려줘" → 이력 조회 후 요약
