# 구글 스프레드시트 자동 연동 설정 가이드

> 이 문서는 컴퓨터에서 구글 스프레드시트를 자동으로 읽고 쓸 수 있도록 **"서비스 계정"**이라는 것을 만드는 과정을 안내합니다.
> 처음 한 번만 설정하면 이후에는 자동으로 동작합니다.
>
> 전체 소요 시간: 약 15~20분

## 1. 구글 클라우드 콘솔 접속 및 프로젝트 만들기

구글 클라우드 콘솔은 구글이 제공하는 **관리자 페이지**입니다. 여기서 자동화에 필요한 설정을 합니다.

### 1-1. 구글 클라우드 콘솔에 접속합니다

- 크롬 브라우저를 열고 주소창에 아래 주소를 입력합니다:
  ```
  https://console.cloud.google.com
  ```
- 구글 로그인 화면이 나오면 **회사에서 사용하는 구글 계정**(Gmail)으로 로그인합니다.

> **주의:** 개인 구글 계정이 아닌 **회사 업무용 구글 계정**으로 로그인해주세요.
> 여러 구글 계정이 있으면 오른쪽 위의 동그란 프로필 사진을 눌러서 올바른 계정인지 확인하세요.

### 1-2. 약관에 동의합니다 (처음 접속하는 경우에만)

- 처음 접속하면 이용약관 동의 화면이 나옵니다.
- **"서비스 약관에 동의합니다"** 체크박스를 클릭하고, 아래쪽의 **"동의 및 계속하기"** 파란 버튼을 클릭합니다.

### 1-3. 새 프로젝트를 만듭니다

- 화면 **맨 위 왼쪽**에 "Google Cloud" 로고 바로 옆에 프로젝트 이름이 보입니다.
  - 처음이면 **"프로젝트 선택"**이라고 적혀 있습니다.
- 그 **"프로젝트 선택"** 글자를 클릭합니다.
- 작은 팝업 창이 뜹니다. 오른쪽 위에 **"새 프로젝트"** 버튼이 있습니다. 이것을 클릭합니다.
- **프로젝트 이름** 입력란에 알아보기 쉬운 이름을 입력합니다:
  ```
  glowseoul-automation
  ```
  (한글은 안 됩니다. 영어, 숫자, 하이픈만 가능합니다.)
- 나머지(조직, 위치)는 그대로 두고 아래의 **"만들기"** 파란 버튼을 클릭합니다.
- 몇 초 기다리면 프로젝트가 생성됩니다.

### 1-4. 방금 만든 프로젝트를 선택합니다

- 오른쪽 위에 알림(종 모양 아이콘) 옆에 **"프로젝트 선택"** 알림이 뜹니다.
- 또는 다시 맨 위 왼쪽의 프로젝트 이름을 클릭하고, 목록에서 방금 만든 **"glowseoul-automation"**을 클릭합니다.

> **확인 방법:** 화면 맨 위 왼쪽에 "glowseoul-automation"이라고 표시되면 제대로 선택된 것입니다.

## 2. 구글 스프레드시트 기능 켜기

기본적으로는 스프레드시트 자동 연동 기능이 꺼져 있습니다. 이것을 켜야 합니다.

### 2-1. 기능 목록 화면으로 이동합니다

- 화면 **왼쪽 위**에 가로줄 세 개 모양(≡)의 **메뉴 버튼**을 클릭합니다.
- 펼쳐진 메뉴에서 아래로 스크롤하여 **"API 및 서비스"**를 찾아 클릭합니다.
- 하위 메뉴가 나오면 **"라이브러리"**를 클릭합니다.

### 2-2. 구글 스프레드시트 기능을 찾습니다

- 화면 가운데에 큰 **검색창**이 있습니다.
- 검색창에 아래 내용을 입력합니다:
  ```
  Google Sheets API
  ```
- 검색 결과에서 **"Google Sheets API"** (구글 스프레드시트 그림이 있는 항목)를 클릭합니다.

### 2-3. 기능을 켭니다

- 상세 화면이 열리면 가운데에 커다란 파란 **"사용"** 버튼이 보입니다.
- 이 **"사용"** 버튼을 클릭합니다.
- 몇 초 후 "사용 설정됨" 또는 "관리" 화면으로 바뀌면 성공입니다.

> **이미 켜져 있는 경우:** "사용" 버튼 대신 "관리" 버튼이 보이면 이미 켜져 있는 것이므로 다음 단계로 넘어가면 됩니다.

## 3. 구글 드라이브 기능 켜기

스프레드시트 파일에 접근하려면 드라이브 기능도 켜야 합니다.

### 3-1. 다시 기능 목록으로 돌아갑니다

- 브라우저의 **뒤로 가기** 버튼을 두 번 클릭하거나,
- 왼쪽 위 메뉴(≡) → **"API 및 서비스"** → **"라이브러리"**를 다시 클릭합니다.

### 3-2. 구글 드라이브 기능을 찾습니다

- 검색창에 아래 내용을 입력합니다:
  ```
  Google Drive API
  ```
- 검색 결과에서 **"Google Drive API"** (구글 드라이브 삼각형 그림이 있는 항목)를 클릭합니다.

### 3-3. 기능을 켭니다

- 2-3 단계와 동일하게 파란 **"사용"** 버튼을 클릭합니다.
- "사용 설정됨" 또는 "관리" 화면이 나오면 성공입니다.

## 4. 서비스 계정 만들기

"서비스 계정"이란 사람이 아닌 **프로그램이 사용하는 전용 계정**입니다. 이 계정을 만들면 프로그램이 자동으로 스프레드시트를 읽고 쓸 수 있게 됩니다.

### 4-1. 사용자 인증 정보 화면으로 이동합니다

- 화면 **왼쪽 위** 메뉴(≡)를 클릭합니다.
- **"API 및 서비스"**를 클릭합니다.
- **"사용자 인증 정보"**를 클릭합니다.

### 4-2. 서비스 계정을 만듭니다

- 화면 **위쪽 중앙**에 **"+ 사용자 인증 정보 만들기"** 버튼이 있습니다. 이것을 클릭합니다.
- 드롭다운 메뉴가 열리면 **"서비스 계정"**을 클릭합니다.

### 4-3. 서비스 계정 정보를 입력합니다

**첫 번째 단계 (서비스 계정 세부정보):**

- **서비스 계정 이름**: 아래 내용을 입력합니다:
  ```
  sheets-automation
  ```
- **서비스 계정 ID**: 이름을 입력하면 자동으로 채워집니다. 그대로 둡니다.
- **서비스 계정 설명** (선택사항): 비워두거나 아래처럼 입력합니다:
  ```
  스프레드시트 자동 연동용
  ```
- 아래쪽의 **"만들고 계속하기"** 파란 버튼을 클릭합니다.

**두 번째 단계 (권한 부여):**

- **"역할 선택"** 드롭다운을 클릭합니다.
- 검색창에 **"편집자"**라고 입력합니다.
- 목록에서 **"기본" → "편집자"**를 클릭합니다.
- 아래의 **"계속"** 버튼을 클릭합니다.

**세 번째 단계 (사용자 액세스 권한):**

- 아무것도 입력하지 않고 **"완료"** 버튼을 클릭합니다.

> **자주 하는 실수:** 두 번째 단계에서 역할을 선택하지 않고 넘어가는 경우가 있습니다.
> 반드시 **"편집자"** 역할을 선택해주세요.

## 5. 인증 키 파일 내려받기

방금 만든 서비스 계정의 **"열쇠"**에 해당하는 파일을 내려받습니다. 이 파일이 있어야 프로그램이 구글에 접속할 수 있습니다.

### 5-1. 서비스 계정 목록에서 방금 만든 계정을 찾습니다

- 4단계가 끝나면 자동으로 "사용자 인증 정보" 화면으로 돌아옵니다.
- 화면 아래쪽 **"서비스 계정"** 목록에 방금 만든 **"sheets-automation"**이 보입니다.
- 그 **이메일 주소**(sheets-automation@glowseoul-automation.iam.gserviceaccount.com 같은 형태)를 클릭합니다.

> **중요:** 이 이메일 주소를 메모장이나 카카오톡 나에게 보내기 등에 **복사해서 저장**해두세요.
> 나중에 7단계에서 다시 필요합니다.

### 5-2. 키 탭으로 이동합니다

- 서비스 계정 상세 화면이 열립니다.
- 화면 위쪽에 여러 탭이 있습니다: **세부정보 / 권한 / 키 / ...**
- **"키"** 탭을 클릭합니다.

### 5-3. 새 키를 만들어 내려받습니다

- **"키 추가"** 버튼을 클릭합니다.
- **"새 키 만들기"**를 클릭합니다.
- 팝업이 나타납니다. **"JSON"**이 선택되어 있는지 확인합니다. (기본값이 JSON이므로 보통 이미 선택되어 있습니다.)
- **"만들기"** 버튼을 클릭합니다.
- 파일이 자동으로 내려받기됩니다. 파일 이름은 `glowseoul-automation-xxxxxx.json` 형태입니다.

> **주의사항:**
> - 이 파일은 **절대로 다른 사람에게 보내거나 공유하면 안 됩니다.** 이 파일이 있으면 누구든 구글 계정에 접근할 수 있습니다.
> - 파일을 잃어버리면 다시 내려받을 수 없습니다. 다시 새 키를 만들어야 합니다.
> - 카카오톡, 이메일 등으로 이 파일을 전송하지 마세요.

## 6. 키 파일을 프로젝트 폴더에 저장하기

내려받은 키 파일을 자동화 프로그램이 찾을 수 있는 위치에 옮깁니다.

### 6-1. config 폴더를 확인합니다

- 맥 파인더(Finder)를 열고 **"문서"** 폴더로 이동합니다.
- **"자동화 work"** 폴더를 엽니다.
- 안에 **"config"** 폴더가 있는지 확인합니다.
  - 없으면 마우스 오른쪽 클릭 → **"새로운 폴더"** → 이름을 `config`로 입력합니다.

### 6-2. 키 파일을 옮깁니다

- 크롬에서 내려받은 파일은 보통 **"다운로드"** 폴더에 있습니다.
- 파인더에서 **"다운로드"** 폴더를 열고, `glowseoul-automation-xxxxxx.json` 파일을 찾습니다.
- 이 파일을 아까 확인한 **config** 폴더 안으로 **드래그(끌어서 놓기)**합니다.
- 파일 이름을 바꿉니다:
  1. 파일을 한 번 클릭한 후 **엔터(Return)** 키를 누릅니다.
  2. 파일 이름을 아래와 같이 변경합니다:
     ```
     google_service_account.json
     ```
  3. **엔터(Return)** 키를 눌러 확정합니다.

> **최종 위치 확인:**
> 파인더에서 아래 경로에 파일이 있으면 정상입니다:
> ```
> 문서 → 자동화 work → config → google_service_account.json
> ```

> **자주 하는 실수:**
> - 파일 이름 끝에 `.json`이 두 번 붙는 경우가 있습니다 (`google_service_account.json.json`).
>   파인더 메뉴에서 **보기 → 파일 확장자 보기**를 켜서 정확히 확인해주세요.
> - 파일 이름에 공백(빈칸)이 들어가지 않도록 주의하세요.

## 7. 스프레드시트에서 서비스 계정에 편집 권한 주기

서비스 계정이 스프레드시트를 읽고 쓰려면, **스프레드시트 공유 설정에서 이 계정에게 편집 권한을 줘야** 합니다.
마치 다른 사람에게 문서를 공유하는 것과 같은 방식입니다.

### 7-1. 자동화할 구글 스프레드시트를 엽니다

- 크롬에서 자동화할 구글 스프레드시트를 엽니다.

### 7-2. 공유 설정을 엽니다

- 스프레드시트 화면 **오른쪽 위**에 초록색 **"공유"** 버튼이 있습니다. 이것을 클릭합니다.

### 7-3. 서비스 계정 이메일을 추가합니다

- "사용자 및 그룹 추가" 입력란이 보입니다.
- 5-1 단계에서 메모해둔 **서비스 계정 이메일 주소**를 입력합니다.
  (예: `sheets-automation@glowseoul-automation.iam.gserviceaccount.com`)
- 오른쪽 드롭다운이 **"편집자"**로 되어 있는지 확인합니다.
  - "뷰어"로 되어 있으면 클릭해서 **"편집자"**로 바꿔주세요.
- **"알림 보내기"** 체크박스가 있으면 **체크를 해제**합니다. (서비스 계정은 이메일을 받을 수 없으므로)
- **"공유"** 또는 **"보내기"** 버튼을 클릭합니다.

> **주의:** 자동화할 스프레드시트가 여러 개면 **각 스프레드시트마다** 이 7단계를 반복해야 합니다.

> **자주 하는 실수:**
> - 이메일 주소를 직접 타이핑하다가 오타가 나는 경우가 많습니다.
>   반드시 **복사하여 붙여넣기** 하세요.
> - 권한을 "뷰어(보기만)"로 두면 프로그램이 데이터를 쓸 수 없습니다. 반드시 **"편집자"**로 설정하세요.

## 8. 자동화 스크립트 실행하기

모든 설정이 끝났습니다. 이제 자동화 프로그램이 잘 동작하는지 확인합니다.

### 8-1. 터미널(명령어 입력 창)을 엽니다

- 맥에서 **Command(⌘) + 스페이스바**를 동시에 누릅니다.
- 검색창에 **"터미널"**이라고 입력합니다.
- **"터미널"** 앱을 클릭하여 엽니다.
- 검은색(또는 흰색) 창이 열립니다.

### 8-2. 프로젝트 폴더로 이동합니다

- 아래 명령어를 그대로 복사하여 터미널에 붙여넣고 **엔터**를 누릅니다:
  ```
  cd ~/Documents/자동화\ work
  ```

### 8-3. 스크립트를 실행합니다

- 아래 명령어를 그대로 복사하여 터미널에 붙여넣고 **엔터**를 누릅니다:
  ```
  python scripts/setup_google_sheets.py
  ```

### 8-4. 결과를 확인합니다

- **성공한 경우:** "연결 성공!" 또는 "설정이 완료되었습니다" 같은 메시지가 나옵니다.
- **실패한 경우:** 빨간 글씨로 오류 메시지가 나옵니다. 아래 "문제 해결" 섹션을 참고하세요.

## 문제 해결

### "파일을 찾을 수 없습니다" 오류

- `config/google_service_account.json` 파일이 올바른 위치에 있는지 확인하세요.
- 파일 이름에 오타가 없는지 확인하세요 (특히 `.json.json` 같은 이중 확장자).

### "권한이 없습니다" 오류

- 7단계에서 스프레드시트 공유 설정을 했는지 확인하세요.
- 공유할 때 **"편집자"** 권한으로 설정했는지 확인하세요.
- 서비스 계정 이메일 주소가 정확한지 확인하세요.

### "API가 활성화되지 않았습니다" 오류

- 2단계(스프레드시트 기능)와 3단계(드라이브 기능)를 모두 켰는지 확인하세요.
- 구글 클라우드 콘솔에서 올바른 프로젝트가 선택되어 있는지 확인하세요 (화면 맨 위 왼쪽 확인).

### 그래도 안 되는 경우

- 터미널 화면을 **캡처(Command + Shift + 3)**하여 담당자에게 보내주세요.
