## 🔥 Phase 6 — 실 환경 검증 및 안정화 (1순위)

### [ ] Task 30. 6개 양식 실 GW 환경 검증 및 필드 매핑 보정
- **대상**: Task 24~29에서 구현한 양식들 (선급금요청/정산, 연장근무, 외근, 증빙발행, 사내추천비)
- **현재 상태**: 코드 구현은 완료했으나 실제 그룹웨어에서 DOM 탐색을 수행하지 않음
- **작업 내용**:
  1. 각 양식을 실제 GW에서 열어 입력 필드의 셀렉터(위치 주소) 확인
  2. 코드와 실제 화면의 차이점 보정
  3. 양식별 최소 1회 보관/상신 테스트 실행
- **예상 소요**: 양식당 1~2세션

### [ ] Task 31. 잔존 좌표 의존 코드 제거 (8개)
- **현재 상태**: 11개 중 3개 교체 완료, 8개 남음 (동적 좌표이나 셀렉터로 교체 가능한 것 포함)
- **대상**:
  - 증빙유형 버튼 (헤더 기준 동적 범위 → 셀렉터 전환 가능)
  - 프로젝트코드도움 더블클릭 (canvas 그리드 → OBTDataGrid API 전환 가능)
  - 세금계산서 모달 체크박스 (좌표 폴백 잔존)
  - 기타 mouse.click 5건
- **목표**: 화면 크기가 바뀌어도 깨지지 않는 안정적 코드
- **선행 조건**: Task 30 일부 병행 가능

### [ ] Task 32. headless 모드 E2E 통합 테스트
- **현재 상태**: 항상 브라우저 화면을 띄워서 테스트 (headed 모드)
- **작업 내용**:
  1. 화면 없이(headless) 1920×1080 해상도에서 T1~T13 전체 실행
  2. 깨지는 항목 식별 및 수정
  3. 배포 환경(서버)에서 구동 가능한 상태로 만들기
- **중요성**: 서버에 올리려면 반드시 headless로 동작해야 함

### [ ] Task 33. 다중 사용자 테스트
- **현재 상태**: tgjeon 계정 1개로만 모든 테스트 수행
- **작업 내용**:
  1. 실 계정 2개 이상으로 로그인 → 결재 → 회의실 예약 테스트
