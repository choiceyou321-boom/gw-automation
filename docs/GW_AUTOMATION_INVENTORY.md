## 5. Export 자동화 — 확보한 셀렉터 + 한계

### 5-1. 확보한 핵심 셀렉터

```python
# GW 모든 페이지의 엑셀 다운로드 아이콘

# 이미지 src 패턴: cel_save_m_normal.{hash}.png
# 좌표 (RM 페이지 기준): (1860, 119)
```

### 5-2. 다운로드 성공 사례

| 파일 | 페이지 | 크기 | 행수 | 컬럼 |
|---|---|---|---|---|
| [v6_CL_MAIN.xlsx](../data/amaranth_exports/v6_CL_MAIN.xlsx) | CL 일정 메인 | 16KB | **162** | 시작/종료/캘린더/제목/내용/등록자/참여자/연락처/장소 |
| [rm_resource.xlsx](../data/amaranth_exports/rm_resource.xlsx) | RM 자원예약 메인 | 7KB | 2 | 일시/종류/자원/예약명/예약자/사용자/내용/상태 |

### 5-3. 자동화의 한계

22개 페이지 URL 직접 진입 결과:

| 결과 | 페이지 수 | 비고 |
|---|---|---|
| 그리드 진입 성공 | 22/22 | 100% URL 진입 가능 |
| 엑셀 아이콘 즉시 보임 | 3/22 | RM/CL/ML만 진입 직후 노출 |
| 엑셀 아이콘 미노출 | 19/22 | BN/HR — **조회 버튼 클릭 후에야 활성**으로 추정 |
| 다운로드 실제 성공 | 2/22 | CL·RM 각 1건 (수동 단일 시도 시) |

### 5-4. 미해결 이슈

1. **조회 버튼 자동 클릭 실패**: BN 보고서 페이지의 데이터 로드 트리거가 페이지별로 다른 셀렉터/조건
2. **다운로드 옵션 모달 미대응**: 엑셀 클릭 후 "포맷/범위 선택" 모달이 뜨면 자동 dismiss 실패 → download event 미발생
3. **6개 모듈 자동 클릭 무반응**: BD/KS/OF/OC/BPM/UT는 자동화 클릭으로 진입 안 됨 (수동은 가능)

## 7. 다음 단계 권장 사항

### 7-1. 분리 계획 v4에 반영
- **GW Adapter Interface**에 모듈 12개 + 페이지 22개 URL 등록
- HR 하위 LNB 트리를 도메인 모델로 표현 (근태/인사/급여/경비/지출/예산)
- 확보한 셀렉터를 `src/shared/gw_session/selectors.py` 같은 상수 모듈로 보관

### 7-2. Export 자동화 다음 시도 (선택)
- **조회 버튼 페이지별 셀렉터 매핑** (각 페이지 1회 사용자 시연 → 매핑 추출)
- **다운로드 옵션 모달 자동 대응** (모달 dismiss 패턴 학습)
- **Anthropic API 크레딧 충전 후 Computer Use** (Vision으로 1마일 돌파)

### 7-3. 현재 자료로 활용 가능한 영역
- **일정 162행** → CRM 미팅 히스토리 분석
- **자원 예약 데이터** → 회의실 사용 패턴 분석
- **HR LNB 트리** → 사내 ERP의 HR 모듈 설계 참고
- **BN 예산 URL** → 예실대비 페이지를 챗봇에서 사용자 GW로 navigate시키는 deep link
