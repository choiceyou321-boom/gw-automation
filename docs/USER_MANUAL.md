# 글로우서울 그룹웨어(WEHAGO) 자동화 비서 - 사용자 매뉴얼

안녕하세요! 글로우서울 그룹웨어 업무를 자동화해주는 **AI 비서** 이용 안내서입니다.
사용자는 **웹 챗봇**과 **텔레그램 메신저** 두 가지 채널 중 편한 곳을 선택하여 이용할 수 있습니다.

지원 기능:
- **회의실 예약 관리** (예약/취소/조회/빈 시간 검색)
- **전자결재 자동 작성** (지출결의서, 거래처등록 등 5종)
- **음성 메시지 인식** (음성→텍스트 변환)
- **계약서 자동 생성** (자재납품/공사 계약서)
- **프로젝트 자금관리** (웹 전용 `/fund` 페이지)
- **AI 인사이트** (프로젝트 리스크/자금 분석)

---

## 1. 웹 서버 챗봇 전용 가이드

PC 환경에서 큰 화면으로 여러 대화 세션을 관리하며 파일 첨부를 편리하게 하고 싶을 때 추천합니다.

### 1.1. 접속 및 초기 설정
1. **접속하기**: 사내망 또는 관리자가 안내한 주소(기본 `http://192.168.0.89:51749`)를 웹 브라우저 주소창에 입력하여 접속합니다.
2. **회원가입**: 화면 중앙의 **[회원가입]** 탭을 클릭한 뒤, 본인의 이름, 직급, 그룹웨어 아이디 및 비밀번호를 입력해 가입합니다.
3. **로그인**: **[로그인]** 탭에서 그룹웨어 아이디와 비밀번호를 입력하여 접속합니다. (최초 로그인 이후에는 브라우저를 닫아도 일정 기간 로그인이 유지됩니다.)

### 1.2. 웹 챗봇 화면 구성 (UI)
- **왼쪽 사이드바 (메뉴)**:
  - **[+ 새 대화]**: 새로운 주제로 대화를 시작합니다.
  - **빠른 실행 버튼**: `회의실 예약`, `지출결의서`, `거래처등록` 등의 버튼을 누르면 요청이 자동 전송됩니다.
  - **프로젝트 관리**: `/fund` 페이지로 이동합니다.
  - **이전 대화 목록**: 과거 대화를 클릭해 이어서 작업할 수 있습니다.
- **오른쪽 상단 메뉴**:
  - **휴지통 아이콘**: 현재 대화방의 내용을 초기화합니다.
  - **[로그아웃]**: 안전하게 로그아웃합니다.
  - *(관리자 전용)* **[관리]**: 구성원 권한 관리 페이지.

### 1.3. 웹 챗봇 사용 방법 및 예시
명령어 규칙 없이 입력창에 **자연스러운 대화**를 입력하거나, 빠른 실행 버튼을 클릭합니다.

#### 파일 첨부 (드래그 앤 드롭)
- 영수증, 통장사본, 사업자등록증, 세금계산서 등의 파일(JPG, PNG, PDF 지원, 파일당 최대 10MB)을 **PC 폴더에서 드래그하여 채팅창 위로 끌어다 놓으면** 즉시 첨부됩니다.
- 입력창 좌측의 클립 아이콘을 클릭해 파일을 직접 선택할 수도 있습니다.
- **AI 자동 인식**: 사업자등록증을 첨부하면 상호, 대표자, 사업자번호 등을 자동 추출합니다. 세금계산서를 첨부하면 거래처, 금액, 날짜를 추출합니다.

#### 음성 파일 전송
- 음성 파일(.ogg, .mp3, .wav, .m4a, .flac 등)을 첨부하면 AI가 자동으로 텍스트로 변환하여 인식합니다.
- 변환된 텍스트 내용에 따라 회의록 정리, 업무 자동화 등 후속 작업을 자동으로 제안합니다.

#### 회의실 예약/조회
- "내일 오후 3시에 2번 회의실 예약해줘. 참석자는 홍길동, 김철수야."
- "오늘 1시부터 2시간 동안 쓸 수 있는 빈 회의실 찾아줘."
- "내일 잡아둔 내 회의실 예약 취소할래."
- "내 예약 목록 보여줘." → 향후 14일 내 본인 예약 목록 표시
- "테스트 예약 정리해줘." → [TEST_] 붙은 예약 일괄 취소

#### 전자결재 자동 작성
- **지출결의서**: "이번 달 야근 식대 2만원 지출결의서 올려줘."
  - AI가 부족한 정보(프로젝트, 금액, 내용)를 단계적으로 질문합니다.
  - 세금계산서 첨부 시 자동으로 거래처/금액/날짜를 추출하여 양식에 반영합니다.
  - **예실대비현황 자동 캡처**: "예산 스크린샷 첨부해줘"라고 요청하면 GW에서 예실대비현황을 자동 캡처하여 첨부합니다.
- **거래처등록**: 사업자등록증 + 통장사본을 첨부한 뒤 "거래처등록 해줘"라고 요청합니다.
- **기타 양식**: 선급금요청, 연장근무, 외근신청 등도 지원합니다.
- **보관(임시저장) 모드**: AI는 서류를 **'임시보관문서'** 폴더에 저장까지만 수행합니다. 내용 확인 후 GW에서 직접 **[상신]** 버튼을 눌러야 결재가 진행됩니다.

#### 결재선 설정
- "내 결재선을 검토:신동관, 승인:최기영으로 설정해줘" → 저장됩니다.
- 양식별로 다른 결재선도 설정 가능합니다.
- 적용 우선순위: ① 대화 중 직접 지정 → ② 양식별 설정 → ③ 기본 설정 → ④ 내장 기본값

#### 계약서 자동 생성
- "자재납품 계약서 만들어줘" 또는 "공사 계약서 작성해줘"라고 요청하면 AI가 단계별로 필요 정보를 질문하고 Word(.docx) 문서를 생성합니다.
- **일괄 생성**: 엑셀 파일(`계약서_입력양식.xlsx`)을 첨부하면 다건 계약서를 한 번에 생성합니다.
- 생성된 계약서는 다운로드 링크를 통해 받을 수 있습니다.

#### 프로젝트 자금관리 (챗봇에서 조회)
- "종로 메디빌더 자금현황 알려줘" → 수주액, 실행예산, 수익금, 하도급 현황 요약
- "프로젝트 비교해줘" → 전체 프로젝트 수금률/지급률/이익률 비교 테이블
- "고수동굴 보고서 만들어줘" → 종합 보고서 생성
- "메디빌더에 A사 하도급 추가해줘, 전기공사 5천만원" → 하도급 업체 추가
- "메디빌더 KOM 완료 처리해줘" → 마일스톤 완료
- "밀린 일 확인해줘" → 기한 초과/미수금/긴급 TODO 조회
- 프로젝트 이름은 부분 입력이 가능합니다 (예: "메디빌더", "고수동굴")
- 상세 관리가 필요하면 "프로젝트 관리 페이지(/fund)에서 확인하세요"라고 안내합니다.

#### 메일 요약
- "새 메일 확인해줘", "메일 요약해줘" → GW 메일함에서 새 메일 확인 및 요약

## 2. 프로젝트 관리 페이지 (`/fund`)

프로젝트별 자금 흐름을 한눈에 파악하고 관리할 수 있는 전용 웹 페이지입니다.

### 2.1. 접속 방법
- 웹 챗봇 로그인 후 사이드바의 **[프로젝트 관리]** 링크를 클릭하거나, 주소창에 `/fund`를 직접 입력합니다.

### 2.2. 프로젝트 관리 (좌측 사이드바)

#### 프로젝트 목록
- 좌측 사이드바에 등록된 프로젝트가 등급별 색상 뱃지와 함께 표시됩니다.
- 프로젝트를 클릭하면 우측에 상세 정보가 표시됩니다.
- **검색**: 사이드바 상단의 검색창에 프로젝트명을 입력하면 필터링됩니다.

#### 새 프로젝트 생성
- **[+ 새 프로젝트]** 버튼 클릭 → 프로젝트명, 등급, 수주액(설계/시공), 실행예산 입력
- 사업코드(GS-25-XXXX 형식)를 입력하면 GW에서 프로젝트 정보를 자동으로 가져올 수 있습니다.

#### 등급 설정
| 등급 | 색상 | 설명 |
|------|------|------|
| **1등급** | 빨강 | 당사 직영(ETF), 확장 가능성 |
| **2등급** | 노랑 | KOM, 2~3차 보고, 결과 보고 |
| **3등급** | 파랑 | KOM, 2차 보고, 결과 보고 |
| **4등급** | 회색 | 기타 |

#### 순서 변경
- 프로젝트를 **드래그**하여 원하는 순서로 정렬할 수 있습니다. 변경된 순서는 자동 저장됩니다.
- 포트폴리오 테이블에서도 드래그 정렬이 가능합니다.

### 2.3. 포트폴리오 (전체 프로젝트 비교)
- 프로젝트를 선택하지 않은 상태에서 **전체 프로젝트 비교 테이블**이 표시됩니다.
- 각 프로젝트의 수주액, 실행예산, 수익금, 이익률을 한눈에 비교할 수 있습니다.
- **AI 분석**: 하단의 "새로 분석하기" 버튼을 클릭하면 AI가 전체 포트폴리오를 분석하여 리스크/액션 요약을 제공합니다.
- **PM 시트 가져오기**: 포트폴리오 화면의 **[PM 시트 가져오기]** 버튼을 클릭하면 Google Sheets(PM팀 Official 시트)에서 하도급 상세, 연락처, 이체내역, 수금현황 데이터를 일괄로 가져와 프로젝트에 반영합니다. 기존에 수기로 관리하던 스프레드시트 데이터를 한 번에 동기화할 때 유용합니다.
- **GW 데이터 동기화** (수동 + 자동):
  - **수동 동기화**: 포트폴리오 화면의 **[전체 GW 동기화]** 버튼을 클릭하면 모든 프로젝트의 GW 데이터를 일괄 업데이트합니다.
    - 3단계로 진행: ① 프로젝트 등록정보 → ② 예실대비현황(사업별) → ③ 예산 합계(상세)
    - 각 단계의 성공/실패 결과가 표시됩니다.
    - 동기화 소요시간: 프로젝트 수에 따라 5~15분
  - **자동 동기화**: 매일 아침 **08:00**에 서버가 자동으로 전체 GW 동기화를 실행합니다. 별도 조작 없이도 출근 시 최신 데이터가 반영되어 있습니다. 자동 동기화 결과는 서버 로그에 기록됩니다.

### 2.4. 탭별 기능 (6개 탭)

프로젝트를 선택하면 6개 탭이 표시됩니다.

#### 대시보드
- **요약 카드 6개**: 수주액 / 실행예산 / 수익금(이익률) / 수금액(수금률) / 하도급업체 수 / 미지급잔액
- **공종별 지급현황 테이블**: 공종명, 업체, 계약금액, 기지급액, 잔액을 한눈에 확인
- **AI 인사이트**: 프로젝트별 분석 결과 카드 (현황 점검, 놓치기 쉬운 리스크, 자금 흐름, 지금 해야 할 일)

#### 개요
- **프로젝트 기본정보**: 카테고리(설계/시공/브랜드/운영/직영/기획 등), 위치, 용도, 면적, 일정(착공일~준공일), 계약현황
- **진행상황 체크리스트**: 마일스톤별 완료 체크
- **배정인원**: 역할별 담당자 목록
- **이슈사항**: 현재 진행 중인 이슈 기록
- **그룹웨어에서 가져오기**: 🌐 버튼을 클릭하면 GW 프로젝트 검색 모달이 열립니다.
  - 키워드로 프로젝트를 검색하여 사업코드를 연결할 수 있습니다.
  - 최초 1회 **[GW에서 프로젝트 목록 가져오기]** 버튼을 눌러 전체 목록을 캐시해야 합니다 (최대 3분 소요).
  - 이후에는 즉시 검색됩니다. 목록 갱신이 필요하면 **[🔄 다시 가져오기]** 버튼을 클릭하세요.
  - 사업코드가 연결되면 GW에서 프로젝트 등록정보 + 예실대비 데이터를 크롤링합니다.

#### 수금현황
- **설계/시공 단계별 수금 관리**: 각 단계의 계약금, 중도금, 잔금 등을 체크박스로 수금 확인
- **실시간 합계**: 총 수금액 / 미수금액 / 수금률을 자동 계산
- 체크박스를 클릭하면 즉시 저장됩니다.

#### 하도급 업체 (서브탭 2개)
- **거래처 연락처**: 업체명, 담당자명, 연락처, 이메일, 메모를 관리합니다. 행 추가/삭제 가능.
- **공종관리**: 공종별 업체 정보와 금액을 관리합니다.

#### 예산&지급관리 (서브탭 2개)
- **하도급 상세**:
  - 업체별 계약금액, 1~4차 지급액 입력
  - 지급확인 체크박스 (체크하면 해당 지급 셀이 연파랑색으로 표시)
  - 기지급액 자동 계산 (1~4차 합계)
  - 잔여금액 자동 계산 (계약금액 - 기지급액)
- **예실대비**:
  - GW에서 크롤링한 예산 대비 실적 현황 (읽기 전용)
  - 예산과목별 예산액 / 집행액 / 잔액 / 대비(%)
  - **연도별 구분**: 전기(2025년)와 당기(2026년) 데이터를 구분하여 표시
  - **합계 카드**: 수입합계, 지출합계, 총잔액을 상단 카드로 표시

#### 이체내역
- GW에서 가져온 이체완료 내역을 조회합니다 (읽기 전용).
- 문서번호, 결의일, 금액, 상태 등이 표시됩니다.

## 3. AI 인사이트 (`/insights`)

프로젝트 데이터를 AI가 분석하여 놓치기 쉬운 포인트를 제시하는 페이지입니다.

### 3.1. 접속 방법
- 프로젝트 관리 페이지 상단 헤더의 **[AI Insights]** 링크를 클릭합니다.

### 3.2. 분석 내용
AI가 프로젝트 데이터를 기반으로 4가지 관점에서 분석합니다:

| 탭 | 내용 |
|----|------|
| **현황 점검** | 수금/지급 진행률, 데이터 누락 확인, 전반적 상태 점검 |
| **놓치기 쉬운 리스크** | 미확인 지급건, 수금 지연, 자금 병목 시점 경고 |
| **자금 흐름** | 수금 vs 지급 타이밍 불일치, 현금 흐름 분석 |
| **지금 해야 할 일** | 구체적인 금액+대상+행동 형태의 실행 권고 |

### 3.3. TODO 관리
- AI 분석 결과에서 누락 데이터가 발견되면 자동으로 `[AI 권고]` 태그가 붙은 TODO가 생성됩니다.
- 직접 TODO를 추가/완료/삭제할 수 있습니다.
- 우선순위(긴급/높음/보통/낮음) 설정이 가능합니다.

### 3.4. 포트폴리오 분석
- 프로젝트를 선택하지 않으면 전체 포트폴리오 종합 분석이 표시됩니다.
- 프로젝트를 선택하면 해당 프로젝트의 분석만 표시됩니다.

## 4. 텔레그램 봇 전용 가이드

언제 어디서나 스마트폰을 통해 그룹웨어 업무를 빠르게 처리하고 싶을 때 추천합니다.

### 4.1. 접속 및 초기 설정
1. **봇 검색 및 시작**: 텔레그램 앱 실행 후 관리자에게 안내받은 봇 아이디를 검색합니다. **[시작]** 또는 `/start` 입력.
2. **회원가입** (2단계 보안):
   - 1단계: `/register [그룹웨어아이디] [이름] [직급]` 입력
   - 2단계: 봇이 비밀번호를 별도 메시지로 요청합니다 → 비밀번호 입력 (입력 후 메시지 자동 삭제)
   - *(이전 방식도 지원)* `/register [아이디] [비밀번호] [이름] [직급]`
3. **로그인** (2단계 보안):
   - 1단계: `/login [그룹웨어아이디]` 입력
   - 2단계: 봇이 비밀번호를 별도 메시지로 요청합니다 → 비밀번호 입력 (입력 후 메시지 자동 삭제)
   - *(이전 방식도 지원)* `/login [아이디] [비밀번호]`

### 4.2. 텔레그램 기본 명령어
| 명령어 | 기능 |
|--------|------|
| `/start` | 봇 시작 및 기본 안내 |
| `/register` | 신규 회원가입 |
| `/login` | 그룹웨어 연동 로그인 |
| `/clear` | **대화 내용 초기화** (AI가 이전 문맥을 혼동할 때 유용) |
| `/setline` | 나만의 **결재선(검토·승인자) 설정** |
| `/myline` | 현재 설정된 결재선 확인 |

### 4.3. 텔레그램 사용 방법 및 예시
대화창에 **자연스러운 대화**로 업무를 지시하세요. 웹 챗봇과 동일한 기능을 사용할 수 있습니다.

#### 회의실 예약/조회
- "내일 오후 3시에 2번 회의실 예약해줘. 참석자는 홍길동, 김철수야."
- "오늘 1시부터 2시간 동안 쓸 수 있는 빈 회의실 찾아줘."
- "내일 잡아둔 내 회의실 예약 취소할래."

#### 전자결재 자동 작성
- "이번 달 야근 식대 2만원 지출결의서 올려줘."
- 영수증 사진이나 PDF를 전송한 뒤 텍스트로 설명해주세요.

#### 음성 메시지
- 텔레그램의 음성 메시지 녹음 기능으로 보내면 자동으로 텍스트 변환 후 처리합니다.
- 음성/오디오 파일을 첨부해도 동일하게 동작합니다.

#### 결재선 설정 (`/setline`, `/myline`)
| 명령어 | 설명 | 예시 |
|--------|------|------|
| 기본 결재선 | 모든 양식 기본값 | `/setline 검토:신동관 승인:최기영` |
| 양식별 결재선 | 특정 양식 전용 | `/setline 지출결의서 검토:신동관 승인:최기영` |
| 간단 결재선 | 검토 없이 승인만 | `/setline 간단 승인:최기영` |
| 확인 | 현재 설정 확인 | `/myline` |

#### 프로젝트 자금관리 (챗봇에서 조회)
- "메디빌더 자금현황 알려줘" → 수주액/실행예산/수익금/하도급 요약
- "프로젝트 비교해줘" → 전체 프로젝트 비교 테이블
- "밀린 일 확인해줘" → 기한 초과/미수금 조회

## 5. 공통 유의사항 및 FAQ

### 보안 관련
- **Q. AI가 예약이나 결재를 최종 승인(상신)까지 바로 해버리나요?**
  - **아닙니다.** 전자결재의 경우, AI는 **'임시보관문서'** 폴더에 저장(보관 모드)까지만 수행합니다. 사용자가 그룹웨어에서 직접 [상신] 버튼을 눌러야 결재가 진행됩니다.

- **Q. 비밀번호를 채팅창에 치는 게 불안해요.**
  - 비밀번호는 AI 엔진으로 전송되지 않으며, 로컬 서버에 Fernet 암호화되어 저장됩니다. 텔레그램에서는 비밀번호 메시지가 자동 삭제됩니다.

### 사용 관련
- **Q. AI가 말을 잘 못 알아듣는 것 같아요.**
  - 대화가 길어지면 문맥 혼동이 생길 수 있습니다. 주제가 바뀔 때는 텔레그램 `/clear`, 웹은 **[+ 새 대화]** 또는 **휴지통** 버튼으로 초기화하세요.

- **Q. 프로젝트 관리 페이지에서 데이터가 안 보여요.**
  - 프로젝트를 선택해야 탭 내용이 표시됩니다. 좌측 사이드바에서 프로젝트를 클릭하세요.

- **Q. GW 동기화가 실패했어요.**
  - GW 세션이 만료되었을 수 있습니다. 웹 챗봇에서 다시 로그인한 후 동기화를 재시도하세요.
  - 동기화는 3단계로 진행되며, 일부 단계가 실패해도 나머지 단계는 정상 진행됩니다.

- **Q. 예실대비 데이터에 전기(이전 연도) 내용이 안 보여요.**
  - 사업코드가 올바르게 입력되어 있는지 확인하세요. 사업별 크롤러가 프로젝트 시작 연도부터 현재까지의 데이터를 자동으로 조회합니다.

- **Q. GW 데이터가 자동으로 업데이트되나요?**
  - **네.** 매일 아침 8시에 서버가 자동으로 모든 프로젝트의 GW 데이터를 동기화합니다. 출근하면 최신 예실대비현황, 이체내역 등이 이미 반영되어 있습니다. 급하게 최신 데이터가 필요한 경우에는 `/fund` 페이지의 **[전체 GW 동기화]** 버튼을 눌러 수동으로도 실행할 수 있습니다.

- **Q. 프로젝트에 AI 분석을 받고 싶어요.**
  - 방법 1: 프로젝트 관리 페이지 대시보드에서 AI 인사이트 카드를 확인합니다.
  - 방법 2: 챗봇에서 "메디빌더 보고서 만들어줘"라고 요청합니다.
  - 방법 3: `/insights` 페이지에서 전체 포트폴리오 AI 분석을 확인합니다.
